Ajax.

AJAX - асинхронний обмін даними з сервером (без перезавантаження сторінки).

Ajax не є самостійною технологією, а лише мостом між java-script та серверним скриптом.

$.ajax({
  type: "POST",
  url: "some.php",
  data: "type=getPosts&number=5&count=3",
  success: function(msg){
    alert( "Відповідь сервера: " + msg );
  }
});

Використовуючи метод POST ми відправляємо на сервер у скрипт some.php запит з такими даними:
type = "getPosts";
number = 5;
count = 3.

На сервері виконається скрипт some.php, він отримає вхідні дані $_POST = [ type: 'getPosts', number: 5, count: 3 ]

Використовуючи ці дані він може виконати певні дії, наприклад:
type: 'getPosts' - звернутися до бази даних та отримати з неї записи;
number: 5 - вибрати записи починаючи з п’ятого;
count = 3 - і вибрати три записа, тобто, 5, 6 та 7.
Отримані записи з бази даних можна повернути назад AJAX-скрипту у вигляді строки тексту чи HTML-коду, XML чи JSON (JavaScript Object Notation).

Наприклад, PHP-скрипт сформує такий масив даних та відправить його назад користувачу:

{ posts: [
  {
    number: 5,
    date: "2016-04-11 08:14:06",
    title: "Сніданок",
    text: "Це мій сьогоднішній сніданок",
    img: "images/IMG0625.JPG",
  },
  {
    number: 6,
    date: "2016-04-11 14:12:06",
    title: "ЗДАВ ЕКЗАМЕН!!!",
    text: "Можете мене привітати, я здав вишку! В п’ятницю ще ОБЖ і я вільний!!",
    img: "images/happy.png",
  },
  {
    number: 7,
    date: "2016-04-11 16:05:24",
    title: "Мій кіт",
    text: "Муркота. Люблю його.",
    img: "images/IMG0628.JPG",
  }
]}

Тепер наша задача зводиться до того, щоб розібрати, розпарсити отримані дані, створити з них HTML-код та вбудувати його в якусь частину нашої сторінки, можливо, видаливши якісь вже неактуальні дані.

В принципі, можна написати PHP-скрипт, що буде присилати вже готовий шмат HTML-коду і наша задача значно спроститься. Але такий підхід зараз рідко використовується. Так само, як з виходом HTML5 остаточно розділилися дані та їх оформлення, так і в цьому випадку - бекенд повинен повертати тільки дані, а фронтенд - формувати з них те, що йому потрібні. І зміна дизайну сайта не повинна зачіпити скрипти на сервері.
