5.1. Робота з jsFiddle.

jsFiddle - пісочниця для веб-девелоперів. Для повноцінної роботи необхідно зареєструватися і залогінитись.

Ця пісочниця являє собою 4 області: для введення блоків коду HTML, CSS, JS і для відображення результату.

Зображення 5.1.1. Інтерфейс сервісу jsFiddle.

Після того, як ви ввели потрібні данні - можна отримати досить коротке посилання на цю роботу, щоб поділитися ним з іншими.

jsFiddle використовують для запитань виду "чого в мене не працює, неправильно відображається", щоб показати реалізацію фрагментів верстки, для спільної роботи над проблемою.

Останнім часом з’явилася можливість спільної роботи - Collaboration.

5.2. CSS: display.

Стиль display може змінити відображення блоку.

Як ми вже знаємо, теги діляться на блочні і строкові (а також табличні, які розглянемо пізніше).

Будь-який блочний елемент можна перетворити на строковий, а будь-який строковий - на блочний за допомогою властивості display.

<span style="display: block;"> - будь-який елемент перетворює на блочний, він набуває всіх властивостей блочних елементів, його ширина по замовчуванню стає 100%, можна застосовувати вертикальні паддінги і маргіни.

<p style="display: inline;"> - будь-який елемент перетворює на строковий, він набуває всіх властивостей строкових елементів, його розміри по замовчуванню визначається розмірами контенту (вмісту), вертикальні паддінги і маргіни будуть ігноруватися.

<div style="display: inline-block;"> - будь-який елемент буде поводитися як строковий, але всередині буде мов блочний - будуть працювати розміри, вертикальні паддінги, але вертикальні маргіни працювати не будуть.

Властивості display: table; та display: table-cell; ми розглянемо пізніше, під час вивчення таблиць. Решта варіантів на практиці використовується дуже рідко і розглядати ми їх не будемо.

<h1 style="display: none;"> - приховує будь-який елемент, так, ніби ви його видалили.

<h1 style="visibility: hidden;"> - приховує будь-який елемент, але залишає виділене під нього місце.

<h1 style="visibility: visible;"> - робить видимим прихований попереднім способом елемент (значення по-замовчуванню).

5.3. Властивості :hover, :active, :visited, :link.

До елементів в стилях можна додати реакцію на події миші (без пробілів):

a:hover { color: green;} - посилання, на яке навели курсор миші.
a:active { color: red;} - посилання, на яке клацнули, але ще не відпустили кнопку миші.
a:visited { color: silver;} - посилання, яке вже відвідали.
a:link { color:orange;} - не відвідуване посилання (аналог простого запису a {...}).
a.first.active:hover {...} - якщо у елемента є класи, то :hover додаємо в кінець.
a#id_element:hover {...} - аналогічно якщо в елемента є ID.
a#first.active:hover {...}, a.active#first:hover {...} - змішані приклади (еквівалентні між собою).
img:hover {...} - стиль буде застосовано, якщо курсор миші навели на зображення.
h2:hover {...} - стиль буде застосовано, якщо курсор миші навели на заголовок.

5.4. Стилі браузера (user agent stylesheet).

По замовчуванню кожен браузер мавласний набір стилів для всіх елементів.

Саме тому "голий" HTML виглядає не як текст в блокноті, а має деяке форматування: заголовки виконані більшим і жирним шрифтом, посилання виділені синім кольором, заголовки і параграфи мають відступи, списки також мають відступи та декоративні кружечки на початку кожного елемента списку.

Зображення 5.4.1. Стилі браузера.

Проблема в тому, що ці стилі у різних браузерах можуть різнитися.

Перед тим, як почати писати стилі бажано провести обнулення стилів браузера.

Ось приклад обнуляючих стилів, включаючи веб-форми.

html { margin:0; padding:0;}
body { margin:0; padding:0; color:#666; font:normal 18px/28px Helvetice, Arial, sans-serif; background:#fff;}
.clear { clear:both;}
p { margin:0; padding:10px 0;}
a { color:#0000ff; text-decoration:underline;}
a:hover { color:#7777ff;}
a img { border:none;}
h1, h2, h3, h4, h5, h6 { margin:0; padding:0; font-weight:bold; color:#000;}
h1 { padding:40px 0 24px; line-height:1.4em;}
h2 { padding:0 0 12px;}
strong { font-weight:bold;}
em { font-style:italic;}
form, menu, iframe { display:block; margin:0; padding:0; border:none;}
ol, ul, dl, li, dt, dd { margin:0; padding:0; list-style:none;}
* { outline:none;}

5.6. ID та вага стилів.

Id - ідентифікатор. Id не може повторюватися в межах однієї сторінки.

div {...} - стилі для всіх тегів.
.class_name {...} - стилі для тегів, що мають цей клас.
#id_name {...} - стилі для тега, який має цей ідентифікатор. Використовувати не бажано.

Вага стилів:
стилі браузера - 0
* - 0.1
тег - 1
клас - 10
id - 100
інлайновий стиль (внутритеговий, індивідуальний) - 1000
!important - 10000

5.6. Акуратність кода.

Акуратний код легко читати та розбирати на окремі елементи.

Всі внутрішні елементи (діти першого порядку) по відношенню до батьківського повинні мати однаковий відступ.

Великі блоки можна відділяти в коді пустими строками для полегшення навігації по коду.

Часто для цього використовують коментарі виду:

<!-- Header begin -->
<div class="header">
  ...
</div>
<!-- Header end -->

Але я вважаю, що вдало названі блоки і задані класи цілком і повністю будуть описувати контент поточного блока.

Принцип бритви Оккама - рос.: "не плоди сущее без необходимости", тобто, не треба створювати зайвих елементів, які не несуть ніякого змісту. Якщо 2 блока можна об’єднати в один - це варто зробити.

Звичайно, не треба прагнути до принципового мінімалізму, часто зменшення HTML-коду збільшує CSS-код і навпаки. Або деякі фрагменти можна зробити економно, але дуже неочевидними способами, в яких важко розібратися, які важко пристосувати до інших умов.

Валідатор - це програма, що перевіряє правильність коду. Основний - це розроблений компанією w3c.

До появи зручних IED (інтегрованих середовищ розробки), наприклад, phpStorm, кожен написаний свій код було дуже бажано закинути у валідатор. Наразі валідатори, як правило, вбудовані в сучасні IDE і перевіряють код на льоту.

Проте, я раджу перші свої проекти закидувати у валідатор для HTML та для CSS щоб ознайомитися як він працює, яку інформацію видає, що пропонує змінити.

Завдання: перевірити свій HTML-код і CSS. Зробіть деякі помили, приберіть атрибути alt у зображень і повторно зробіть перевірку.
